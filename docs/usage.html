Milestone Planner 使い方

「中長期のマイルストーン」を CSV で管理し、ブラウザで一覧とタイムライン表示を行うためのツールです。

← ツール本体に戻る

1. このツールでできること

  • マイルストーン（大目標〜小タスク）を CSVファイル で管理します。
  • ブラウザ上で リスト表示 と タイムライン表示 ができます。
  • 状態・担当者・期間などを編集し、再び CSVとして書き出し できます。
  • 処理はすべてブラウザ内で完結し、 サーバには送信されません。

2. フォルダ構成と「data フォルダ」

想定しているフォルダ構成の例です。

milestone-tool/
├─ index.html            # ツール本体
├─ style.css             # 見た目の設定
├─ script.js             # ロジック
├─ README.md
├─ sample/
│   └─ milestone_sample.csv   # サンプルデータ
├─ docs/
│   └─ usage.html        # このページ
└─ data/                 # ★ 実際の運用用CSVを置くフォルダ

ブラウザから data フォルダに直接書き込むことはできません。
書き出し時に表示される「名前を付けて保存」ダイアログで、data フォルダを選んで保存してください。

3. CSV フォーマット

CSV のヘッダ行は次の通りです。

id,title,level,category,start_date,end_date,status,priority,owner,tags,notes

主な列の意味

  • id : 任意のID（例: ms-001）。空でも可。
  • title : マイルストーン名（必須）。
  • level : L1 L2 L3 （大目標 / 中目標 / 小タスク）
  • category : 分類（例：研究 / 教育 / 運営 / 私生活 など）。
  • start_date, end_date : 日付（YYYY-MM-DD）。
  • status : not-started / in-progress / done / postponed
  • priority : low / medium / high
  • owner : 担当者名やイニシャル（例：Kaneko）。
  • tags : セミコロン区切りのタグ（例： 2025計画;科研費;教育）。
  • notes : メモ。カンマを含む場合は "..." で囲みます。

まずは sample/milestone_sample.csv をコピーして編集すると、項目漏れが少なくて済みます。

4. 使い始める手順（最短ルート）

4.1 サンプルCSVから作る

  1. sample/milestone_sample.csv をコピーします。
  2. Excel や Google スプレッドシートで開き、タイトルや日付などを書き換えます。
  3. 文字コード UTF-8 の CSV として保存します。
  4. data フォルダを作り、その中に保存しておくと整理しやすくなります。

4.2 ツールを開く

  1. milestone-tool フォルダ内の index.html をブラウザで開きます。
  2. Milestone Planner の画面が表示されます。

4.3 CSV を読み込む

  1. 画面左上「CSV管理」で、作成した CSV ファイルを選択します。
  2. 読み込みモードを選びます：
    • 置き換え：既存データをすべて消して、このCSVで上書き。
    • 追加：既存データの末尾に追加。
  3. 「CSV読み込み」ボタンを押すと、件数が表示され、リストとタイムラインが更新されます。

5. 画面の見方と編集

5.1 リスト表示

  • 「リスト表示」タブでは、マイルストーンが表形式で表示されます。
  • 行をクリックすると、右側の「詳細」パネルに内容が表示されます。
  • ヘッダ（Lv / タイトル / 期間 など）をクリックすると、その列でソートできます。

5.2 詳細パネルからの操作

  • 編集：モーダルが開き、内容を修正できます。
  • 複製して新規：同じ内容のマイルストーンを新しい ID で追加します。
  • 削除：確認のうえ、マイルストーンを削除します。

5.3 タイムライン表示

  • 「タイムライン表示」タブでは、横軸が日付、縦方向がカテゴリごとのレーンになります。
  • レベル（L1 / L2 / L3）によって色が変わり、全体の流れが一目でわかります。
  • バーをクリックすると、詳細パネルが更新されます。

5.4 フィルタ

  • キーワード、レベル、状態、担当者で絞り込みができます。
  • キーワードは、タイトル・カテゴリ・タグ・メモなどをまとめて検索します。

6. CSV に保存する（上書きのイメージ）

  1. 画面左上「CSV書き出し」ボタンをクリックします。
  2. 対応ブラウザ（Chrome / Edge など）の場合：
    • OS の「名前を付けて保存」ダイアログが開きます。
    • data フォルダを選び、ファイル名を確認します（初期値：milestones_YYYY-MM-DD.csv）。
    • 元の CSV と同じ名前にすれば「上書き保存」に近い運用ができます。
  3. Safari / Firefox などでは：
    • ファイル名を入力するダイアログが表示されます。
    • その後、通常のダウンロードが行われます（保存先はブラウザの設定に依存）。

ツール上で編集した内容は、自動では元のCSVに反映されません。
変更を残したい場合は、必ず「CSV書き出し」で保存してください。

7. よくある質問

Q. オフラインでも使えますか？

A. はい。フォルダ一式をローカルに置けば、インターネット接続なしで動作します。

Q. データはどこかのサーバに送信されますか？

A. いいえ。CSVの読み込み・編集・書き出しはすべてブラウザ内で完結します。

Q. 列を増やしたい場合は？

A. 列を追加したい場合は、script.js 内の読み込み・書き出し処理の修正が必要です。
まずは標準の11列に慣れてから拡張することをおすすめします。
